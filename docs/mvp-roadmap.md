# MVP Roadmap

## Phase 0

- establish repo structure
- keep shared logic package-oriented
- avoid framework lock-in before the API contract is settled

## Phase 1

- define the client-side auth flow against the future iWatched scrobble API
- implement the shared API client package
- define adapter interfaces
- scaffold the browser extension app

## Phase 2

- support a narrow first adapter set
- add confirmation UX for low-confidence matches
- add local queueing and retry behavior

## Phase 3

- harden telemetry and debugging flows
- expand supported platforms
- decide whether a desktop companion is justified
